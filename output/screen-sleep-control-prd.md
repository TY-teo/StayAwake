# ScreenSleepControl - 产品需求文档 (PRD)

> Super Dev | 阶段 2/9 docs | 主导专家: PM
> 继承自: screen-sleep-control-research.md

## 1. 产品定位

一个常驻 macOS 菜单栏的轻量工具，让用户用最少的点击控制两件事：系统不因空闲而休眠、合盖后任务持续运行。界面采用 macOS 26 液态玻璃风格，操作直接发生在菜单栏弹出的玻璃面板上。

- 产品代号：ScreenSleepControl（工程名）；显示名已确认为 **"StayAwake"**
- 已确认决策：合盖特权采用 **方案 A 管理员授权弹窗**；**MVP 包含定时自动关闭**（30m/1h/2h/5h/自定义）
- 平台：macOS 26.0+（Apple Silicon 与 Intel 均可，Liquid Glass 需 macOS 26）
- 形态：菜单栏 App（无 Dock 图标，LSUIElement = true）
- 用户：需要长任务持续运行的个人开发者/创作者（本人自用）

## 2. 用户故事

- US-1：作为用户，我点一下菜单栏图标里的"保持唤醒"开关，机器就不再因空闲休眠，无论是否插电。
- US-2：作为用户，我可以选择"屏幕也常亮"或"允许屏幕关闭但系统不休眠"（跑任务时黑屏省电）。
- US-3：作为用户，我打开"合盖继续运行"开关后，合上盖子机器仍在跑任务。
- US-4：作为用户，开启合盖模式前能看到散热/电量风险提示，避免误用。
- US-5：作为用户，关闭开关或退出 App 时，系统电源行为自动恢复正常，不残留。
- US-6：作为用户，我能从菜单栏图标的样子直接看出当前是否处于"保持唤醒"和"合盖运行"状态。
- US-7：作为用户，我可以设置"保持唤醒"的时长（一直保持 / 指定时长后自动关闭）。
- US-8：作为用户，我可以设置开机自动启动该工具。

## 3. 功能需求

### F1 保持唤醒（核心，无需特权）

- F1.1 主开关：开/关系统空闲休眠阻止（电源断言）。
- F1.2 屏幕策略子选项：
  - 屏幕也常亮（同时阻止显示器休眠）。
  - 仅系统唤醒，允许屏幕关闭（默认推荐，省电）。
- F1.3 时长控制：一直保持（默认）/ 预设时长（30 分钟、1 小时、2 小时、5 小时）/ 自定义；到时自动关闭并恢复。
- F1.4 状态持久化：记住用户上次的屏幕策略与默认时长偏好。

### F2 合盖继续运行（核心，需特权）

- F2.1 独立开关：开启后执行 `pmset -a disablesleep 1`，关闭后执行 `pmset -a disablesleep 0`。
- F2.2 风险确认：首次开启时弹出说明（散热/电量风险、建议插电），用户确认后才执行。
- F2.3 自动还原：关闭开关、退出 App 时还原为 `0`；下次启动自检，如检测到由本 App 设置的残留则还原。
- F2.4 特权获取：按选定方案（管理员弹窗 / 特权 helper）获取执行权限。

### F3 菜单栏交互（核心）

- F3.1 常驻菜单栏图标，使用 SF Symbols，按状态切换图标与着色：
  - 默认（允许休眠）：休眠态图标。
  - 保持唤醒中：唤醒态图标。
  - 合盖运行中：叠加合盖态标识/着色。
- F3.2 点击图标弹出液态玻璃面板，承载两个主开关、屏幕策略、时长、状态行与次级入口。
- F3.3 面板内开关即点即生效，并即时反馈状态文案与图标。

### F4 偏好与系统集成（增强）

- F4.1 开机自启（`SMAppService.mainApp` 登录项注册）。
- F4.2 退出按钮、关于信息、当前生效状态摘要。
- F4.3 当系统"减弱透明度"开启时，UI 自动降级为不透明材质仍可用。

## 4. 范围边界

纳入 MVP：F1 全部、F2 全部、F3 全部、F4.1/F4.2/F4.3。
不在本期：触发器/自动化规则（如"某 App 运行时自动保持唤醒")、多显示器策略、菜单栏外的独立窗口、iCloud 同步、本地化多语言（首发中文，文案预留可扩展）。

## 5. 验收标准（关键）

- AC-1：开启"保持唤醒"后，空闲超过系统原休眠时间仍不休眠（插电与电池两种情况均验证）。
- AC-2：选择"允许屏幕关闭"时，屏幕可正常息屏但系统继续运行。
- AC-3：开启"合盖运行"并合盖后，任务进程持续运行；关闭开关后合盖恢复正常休眠。
- AC-4：退出 App 后 `pmset -g | grep -i disablesleep` 显示已恢复为 0；电源断言全部释放（`pmset -g assertions` 无本 App 残留）。
- AC-5：菜单栏图标状态与实际电源行为一致。
- AC-6：开启"减弱透明度"后面板仍清晰可用。
- AC-7：构建零错误、零警告；运行无控制台红色错误。

## 6. 非功能需求

- 性能：常驻内存占用低（目标 < 50MB），空闲 CPU 近 0。
- 安全：特权操作最小化，仅 `pmset disablesleep` 一项；不硬编码任何凭证；管理员授权由系统弹窗处理。
- 可靠性：任何异常路径都不得让系统永久卡在不休眠状态（多重还原兜底）。
- 可访问性：开关具备 accessibility label；尊重"减弱透明度"。
